## 1) 页面与组件对应

### 首页（Home）
- **导航栏**：`Navbar`
  - **现状**：Logo 里已有 `Q` 字样（`<div ...>Q</div>`），搜索框占位符为“搜索材料...”
  - **需求**：在“顶部搜索材料的旁白”加一个 **Q 标识**，作为**管理员登录入口**（点击后进入登录页/弹窗，成功后进入后台）
- **左侧 4 个浮动图标**：`SidebarSocial` + `ContactModal`
  - **现状**：4 个社交 icon（Facebook/Twitter/Linkedin/Instagram）点击都打开同一个 `ContactModal`（电话/邮箱/WhatsApp）
  - **需求**：点击弹窗的信息需可在后台配置，且后台模块与前台对齐
- **“需要专业材料咨询？”模块**：`HomePage` 的 “Consultation Box” 区块
  - **现状**：
    - 背景：`bg-white/60 backdrop-blur-xl`，内部热线“+86 21 ...”
    - 按钮“在线预约咨询”打开 AI 弹窗（`isAIChatOpen`）
  - **需求**：
    - 背景透明度需调整（更透，避免遮挡背后文字）
    - “咨询热线”改为 **WhatsApp 联系方式**（后台可配置）
    - 在线预约弹窗接入 **DeepSeek** + 工具调用（产品卡片/算量/一键下单）
    - 增加 **AI 咨询监控模块**（后台查看 IP + 关键信息 + 沟通记录）
- **核心系统解决方案**：`SYSTEMS` 数据 + `HomePage` “Core Products Section” 列表
  - **需求**：后台可改文案/图片（与前台对齐）
- **经典工程案例**：`PROJECTS` 数据 + `HomePage` “Projects Preview Section”
  - **需求**：后台可改文案/图片（与前台对齐）
- **全球物流与追踪**：`HomePage` “Logistics Section”
  - **需求**：后台可改文案/图片（与前台对齐）
- **底部 Footer**：`Footer`
  - **需求**：后台可改文案/图片/联系方式（与前台对齐）

### 产品目录（Catalog）
- **目录页组件**：`CatalogPage`
  - **现状**：
    - 产品来自前端常量 `PRODUCTS`
    - 分类 UI 为固定数组（“全部材料/轻钢龙骨/石膏板/...”），未与 `PRODUCTS` 强绑定
    - 点击产品标题会 `setActiveProduct(product)` 并进入详情页 `detail`
    - “需要定制规格？联系定制”按钮目前无后台配置入口
  - **需求**：
    - 后台要有产品/分类/子分类的完整 CRUD（增删改查、上下架/价格、上传照片、规格型号等）
    - “联系定制”弹窗联系方式后台可改

### 产品详情（Detail）
- `ProductDetailPage`
  - **现状**：只支持数量 `qty`，点击“加入采购清单”调用 `onAddToCart(product)`（一次加 1）
  - **需求联动**：AI “一键下单”要能以（产品、数量）加入购物车（需扩展加入购物车接口/状态）

### 购物车（Cart）
- `CartPage`
  - **现状**：
    - 物流与联系信息字段：`port`（单个 select，固定 4 个港口选项）、`method`（EXW/FOB/CIF/DDP）、`contact.email`、`contact.social`
    - 费用结算：使用人民币符号 `¥`，且 `shipping/tax/total` 为前端假规则（与产品尺寸无关）
    - 提交按钮仅 `alert()`，未落库
  - **需求**：
    - 目的港：**先选国家，再选港口**（港口数据后台维护/录入）
    - 必填 gating：采购方式/联系邮箱/社交账户/（第 4 个必填字段待落地，建议：联系人姓名或电话）缺一不可 → 不显示费用结算模块
    - 费用结算：**美元 USD**；按货品“大小”计算，采用 **CBM（m³）** 模型
    - 后台记录：用户填写信息 + 采购清单 + 费用明细，管理员可查看

### AI 咨询弹窗（在线预约咨询）
- `AIChatModal`
  - **现状**：前端 `setTimeout` 模拟 AI，按 `PRODUCTS.find` 命中产品名后显示产品卡片
  - **需求**：
    - 真实接入 DeepSeek（密钥放后端环境变量，禁止前端暴露）
    - 提示词：销售顾问角色（已在需求文件给出）
    - 工具能力：产品检索/推荐、算量（参考投喂 HTML 计算器逻辑）、一键下单（加购到购物车）
    - 监控落库：IP、核心问题摘要、会话记录

## 2) 后台页面清单（建议）

### 登录与基础
- 管理员登录（账号 QT123 / 密码 QT123 测试）
- 会话管理（JWT + 后端校验）

### CMS 配置（与首页/浮窗对齐）
- 顶部导航：管理员入口开关/样式（Q 标识位置）
- 左侧浮动图标与弹窗：图标/标题/字段（电话/邮箱/WhatsApp 等）
- 首页咨询模块：背景透明度、WhatsApp 联系方式、弹窗文案
- 核心系统解决方案：列表项（标题/描述/图片/features）
- 工程案例：列表项（标题/地点/描述/图片）
- 物流与追踪：标题/描述/卡片文案/图片
- Footer：地址/电话/邮箱/外链/版权

### 产品与分类
- 分类 CRUD（支持排序/启用）
- 子分类 CRUD（隶属分类）
- 产品 CRUD（名称/分类/子分类/价格 USD/状态/规格/描述/图片）
- 产品尺寸/体积字段（用于 CBM 结算）：长/宽/高（单位统一）或 `cbm_per_unit`

### 港口与费率
- 国家 CRUD
- 港口 CRUD（挂在国家下）
- 运费规则（按港口）：`usd_per_cbm`、最低计费 cbm（可选）、固定附加费（可配置 JSON）

### 订单/采购申请
- 订单列表/详情：联系信息、目的国/港口、采购方式、购物车清单、CBM、费用明细、提交时间、IP

### AI 咨询监控
- 会话列表/详情：IP、时间、摘要、原始问答、推荐/加购事件
